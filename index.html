What is AES?
🔐
The Big Picture — What AES does and why it matters

AES (Advanced Encryption Standard) is the encryption algorithm used by virtually everything on the internet today — WhatsApp, HTTPS, your bank, disk encryption, and more. It was standardized by NIST in 2001 and replaced the older DES.

Substitution-Permutation

Uses a network of substitution (S-Boxes) and permutation (mixing) — unlike DES's Feistel structure.

All 4 AES operations — SubBytes, ShiftRows, MixColumns, AddRoundKey — work directly on this State grid. The grid is your "workspace" for the entire encryption. At the end, bytes are read back out column by column to form the ciphertext.
The 4 Operations

Each of AES's 10 rounds applies exactly these 4 operations in order. Let's understand each one deeply before seeing them run live.

①

SubBytes Confusion

Every byte in the 16-byte State is independently looked up in the AES S-Box — a fixed 256-entry substitution table — and replaced with its corresponding output byte. This is the only non-linear step, and it's what makes AES impossible to crack with linear algebra.

②

ShiftRows Diffusion

Each row of the State grid is rotated left by a different number of positions: Row 0 stays, Row 1 rotates 1 left, Row 2 rotates 2 left, Row 3 rotates 3 left. Simple, but it ensures bytes from every column end up in every other column.

③

MixColumns Diffusion

Each column of 4 bytes is multiplied by a fixed mathematical matrix using Galois Field (GF(2⁸)) arithmetic. Every output byte depends on all 4 input bytes of that column — creating maximum interdependency. This is where the real mixing happens.

④

AddRoundKey Key Mixing

Each bit of the State is XOR'd with the corresponding bit of the round key. This is where your secret key actually enters the computation. Without knowing the key, you can't reverse this step.

①
SubBytes — The S-Box Substitution (Deep Dive)

The S-Box converts every byte to a different byte using a mathematically constructed lookup table. It's the heart of AES security.

? How was the S-Box built? (The underlying math)

The S-Box is not random — it was mathematically designed to have the best possible resistance to attacks. Each input byte goes through exactly two steps:

Step A — Multiplicative Inverse in GF(2⁸)

For input byte b, find the value b⁻¹ such that b × b⁻¹ = 01 in GF(2⁸) arithmetic. Special case: 0x00 maps to itself.

Step B — Affine Transformation

Apply a bit-rotation matrix and XOR with constant 0x63. This breaks algebraic structure and prevents mathematical pattern attacks.

✨
Good news: The S-Box is pre-computed — you just look up the table! You never calculate this during actual encryption. The math involved behind can be learnt from the video below.
Full AES S-Box — 256 entries (input byte → output byte)

Row = high nibble of input, Column = low nibble. E.g. for input 0x53: row=5, col=3 → output = 0xED

🛡️
Why S-Box = security? Without SubBytes, AES would be linear — meaning an attacker could solve it with a system of equations. The S-Box's non-linearity makes that mathematically impossible. Even a 1-bit change in input flips 4–5 output bits unpredictably.
🧮
Galois Field GF(2⁸) — The Special Math Behind AES

AES uses Galois Field arithmetic GF(2⁸) in two places: building the S-Box and in MixColumns. It sounds intimidating, but the idea is straightforward.

💡 The Core Idea: Math that stays within 8 bits, always

Normal integer arithmetic lets numbers grow without bound (3 × 100 = 300). GF(2⁸) forces every result to stay within 0–255 (one byte) — which is perfect for byte-by-byte cryptography.

Addition in GF(2⁸)

Just XOR the bits. No carrying, no overflow. Simple!

0x57 ⊕ 0x83 = 0xD4
0101 0111
1000 0011
─────────
1101 0100
Multiplication in GF(2⁸)

Multiply like polynomials, then reduce modulo the AES irreducible polynomial to keep result in 8 bits.

Modulus = x⁸+x⁴+x³+x+1 = 0x11B

Wide Trail Strategy: MixColumns was designed so that any 4-round trail has at least 25 active S-Boxes. This guarantees that differential and linear attacks are computationally infeasible — AES has provable security bounds built into its structure.
④
AddRoundKey — Where Your Secret Key Enters

The simplest operation — XOR every bit of the State with the corresponding bit of the round key. This is the only step that uses the key.

State[row][col] = State[row][col] ⊕ RoundKey[row][col]
// Example — first byte only:
State[0][0] = D4 (current state value)
RoundKey[0][0] = 63 (byte from round key)
Result :
D4 -> 1011 0100
63 -> 0110 0011
---------------
B7 -> 1011 0111 // This happens for all 16 bytes simultaneously
🔑
AddRoundKey timing: It runs before round 1 (called initial key whitening), and then at the end of every round. For AES-128 this means the key mixes in 11 times total across the 10 rounds. Without this step, AES would be keyless — a useless permutation anyone could reverse!
Key Expansion
🗝️
How AES generates 11 different round keys from your 128-bit key

AES-128 needs 11 round keys (one initial + one per round). The Key Expansion algorithm generates all of them from your original 128-bit key using a clever word-based process.

📐 The Algorithm — Words and Transformations

A "word" = 4 bytes = one column of the State. Your 128-bit key = 4 words (W[0]–W[3]). Each subsequent word is built from the previous:

        // AES-128 Key Expansion
        // W[0]..W[3] = original key
        // Generate W[4] to W[43]
        for (int i = 4; i < 44; i++) {
            temp = W[i - 1];
            if (i % 4 == 0) {
                temp = RotWord(temp);      // [a,b,c,d] → [b,c,d,a]
                temp = SubWord(temp);      // Apply S-box to each byte
                temp = temp ^ Rcon[i / 4]; // XOR with round constant
            }
            W[i] = W[i - 4] ^ temp;        // XOR with word 4 positions back
        }
        // Each round key consists of 4 words:
        // Round k → W[4k], W[4k+1], W[4k+2], W[4k+3]

Why RotWord + SubWord + Rcon? These prevent weak key schedules where similar keys produce similar round keys. The non-linearity of SubWord and the different Rcon per round guarantee all 11 round keys are maximally different from each other.

Why AES is Secure
🛡️
Security Analysis — Three Properties That Make AES Unbreakable
🌫️
Confusion

SubBytes makes the relationship between key and ciphertext extremely complex. Changing 1 key bit changes ~50% of the ciphertext — unpredictably.

🌊
Diffusion

ShiftRows + MixColumns spread each input bit across all 128 output bits within 2 rounds. Every output bit depends on every input bit.

🔑
Key Schedule

11 well-diffused round keys, each derived through SubWord + RotWord + Rcon. While generated from the same original key, they are designed so that no simple relationships exist between them.

// Fastest supercomputer in the world: ~10¹⁸ operations/sec

Still essentially brute force — no significant algebraic weakness found after 25+ years of cryptanalysis by the world's best mathematicians
